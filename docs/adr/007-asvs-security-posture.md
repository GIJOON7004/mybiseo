# ADR-007: OWASP ASVS 보안 수준 결정

**상태:** 승인 (2026-05-10)
**작성자:** Manus AI
**검증 기준:** OWASP ASVS v4.0.3 원문 ([GitHub](https://github.com/OWASP/ASVS/tree/master/4.0/en))

---

## 맥락

3개 외부 AI 코드리뷰(제미나이, 젠스파크, 클로드)에서 OWASP ASVS Level 3 적용을 제안했습니다. ASVS v4.0.3은 총 286개 보안 요구사항을 정의하며, Level 3은 금융/군사/의료 EMR 등 고위험 시스템 대상입니다. 마이비서의 적정 보안 수준을 결정합니다.

## 결정

**ASVS Level 1을 전수 적용하고, Level 2에서 V7(Logging), V8(Data Protection), V14(Configuration) 3개 챕터를 선별 적용합니다.** Level 3 전체 286개 요구사항은 적용하지 않습니다.

## 근거

마이비서는 환자 PHI(Protected Health Information)를 직접 저장하지 않는 SEO 진단 SaaS입니다. 그러나 병원장 PII(이메일, 전화번호)와 병원 경영정보(매출 추정치, 경쟁사 분석)를 다루므로 Level 1만으로는 부족하고, Level 3 전체는 과잉입니다.

## 재검토 트리거

다음 조건 중 하나라도 충족되면 ASVS 수준을 재평가합니다:

1. **환자 데이터 저장 기능 추가** — Level 3 전체 적용 필수
2. **팀 규모 3인 이상** — 감사 로그 테이블 + 시크릿 로테이션 구현
3. **인수실사 진행** — 미통과 5개 항목 사전 해결
4. **개인정보 유출 사고 발생** — 전체 보안 재감사
5. **서비스 정식 공개** — 8.3.2(데이터 삭제), 8.3.8(보존 정책) 필수 해결

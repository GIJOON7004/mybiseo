# ADR-005: 테스트 스냅샷 갱신 권한 및 골든 테스트 정책

## Status
Accepted (2026-05-10)

## Context

SEO 진단 결과의 일관성을 보장하기 위해 골든 테스트 세트(Golden Test Set)를 운영한다. 진단 알고리즘 변경 시 스냅샷이 깨질 수 있으며, "의도된 변경"과 "회귀 버그"를 구분하는 절차가 필요하다.

## Decision

### 스냅샷 갱신 절차

1. **자동 갱신 금지:** `pnpm test -- --update`를 직접 실행하지 않는다.
2. **변경 사유 기록:** 스냅샷이 깨지면 해당 commit message에 `[snapshot-update]` prefix와 변경 사유를 명시한다.
3. **검증 체크리스트:**
   - 점수 변동이 ±5점 이내인지 확인
   - 카테고리 만점 초과가 발생하지 않는지 확인
   - 등급(A~F) 변동이 1단계 이내인지 확인

### 골든 테스트 세트 구성

| 테스트 사이트 | 특성 | 기대 등급 |
|--------------|------|-----------|
| gijoon.com | 일반 병원 사이트 | B~C |
| SPA 사이트 (렌더링 실패) | 진단 한계 케이스 | F + 한계 고지 |
| 대형 병원 (100+ 페이지) | 크롤링 성능 | A~B |

### 갱신 권한

- **AI 에이전트 (Manus):** 스냅샷 갱신 가능하나, 반드시 commit message에 사유 명시
- **향후 개발자:** PR description에 before/after 점수 비교표 첨부 필수

## Consequences

- 진단 알고리즘 변경 시 스냅샷 깨짐 → CI 실패 → 의도적 변경임을 증명해야 merge 가능
- 무의식적 회귀 방지 (점수 산정 로직 실수 즉시 탐지)

## Alternatives Considered

1. **스냅샷 테스트 미사용:** 채택하지 않음. 진단 결과 일관성이 서비스 신뢰도의 핵심.
2. **매 커밋마다 실제 사이트 크롤링:** 채택하지 않음. 외부 의존성으로 CI 불안정.

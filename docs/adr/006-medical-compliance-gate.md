# ADR-006: 의료법 준수 게이트 (Medical Compliance Gate)

## Status
Accepted (2026-05-10)

## Context

마이비서는 병원/의료기관 대상 SEO 서비스를 제공한다. AI가 생성하는 블로그 콘텐츠, 광고 문구, 소셜 미디어 포스트가 의료법 제56조(의료광고 제한)를 위반할 경우 고객이 법적 제재를 받을 수 있다. 이는 서비스 신뢰도와 직결되는 핵심 리스크이다.

## Decision

콘텐츠 생성 파이프라인에 **Medical Compliance Gate**를 필수 단계로 삽입한다.

### 게이트 위치

```
[AI 콘텐츠 생성] → [Medical Compliance Gate] → [사용자 검토] → [발행]
```

### 검증 규칙 (의료법 제56조 기반)

| 규칙 ID | 검증 항목 | 위반 예시 |
|---------|-----------|-----------|
| MED-001 | 치료 효과 보장 표현 금지 | "100% 완치", "반드시 효과" |
| MED-002 | 비교 광고 금지 | "타 병원보다 우수" |
| MED-003 | 환자 치료 전후 사진 제한 | 동의 없는 before/after |
| MED-004 | 가격 할인 유인 제한 | "50% 할인 이벤트" |
| MED-005 | 의료인 경력 과장 금지 | 검증 불가능한 "최초", "유일" |

### 구현 방식

1. **LLM 기반 1차 검증:** 콘텐츠 생성 후 별도 LLM 호출로 의료법 위반 여부 판단
2. **키워드 기반 2차 검증:** 금지 표현 사전(regex)으로 하드코딩 검증
3. **위반 시 동작:** 콘텐츠 발행 차단 + 사용자에게 위반 사유 및 수정 제안 표시

### 게이트 통과 기준

- LLM 검증: `compliance_score >= 0.8` (1.0 만점)
- 키워드 검증: 금지 표현 0개

## Consequences

**긍정적:**
- 고객 법적 리스크 사전 차단
- 서비스 차별화 (경쟁사 대비 법적 안전성 보장)
- 인수실사 시 "법적 리스크 관리 체계" 증빙

**부정적:**
- 콘텐츠 생성 시간 ~2초 추가 (LLM 2회 호출)
- False positive로 정상 콘텐츠 차단 가능 → 사용자 수동 override 옵션 제공

## Alternatives Considered

1. **사용자 자가 검증만:** 채택하지 않음. 비전문가 고객이 의료법 숙지 불가.
2. **외부 법률 검토 API:** 존재하지 않음. 한국 의료법 특화 서비스 부재.
3. **발행 후 모니터링:** 채택하지 않음. 위반 발생 후 조치는 이미 늦음.

## 즉시 수정 (영업 신뢰도 직결)

### 1. "월 월" 중복 — 변수 치환 버그
- 위치: `server/ai-visibility-report.ts` 1134행
- 원인: `revenueImpact`에 이미 "월 약 3,000만원" 포함 → 1134행에서 다시 "월" 추가
- 수정: numMatch 추출 시 "월"이 이미 있으면 추가하지 않도록 조건 변경

### 2. "흥들리는지" → "흔들리는지"
- 위치: `server/seo-analyzer-v4-items.ts` 769행
- 수정: 단순 오타 수정

### 3. "연간 1억 4,400만원" 하드코딩 의심
- 원인: LLM(closingStatement)이 자체적으로 "연간 약 1억 4,400만원" 생성
- reality-diagnosis.ts에서 closingStatement 내 매출 관련 수치를 동적 계산값으로 치환하는 로직이 있음
- 하지만 "연간" 패턴은 치환 대상에 포함되지 않음
- 수정: closingStatement 후처리에 "연간.*만원" 패턴도 동적 계산값으로 치환 추가

### 4. "웹사이트 유입 누락 환자" 표현 개선
- 위치: reality-diagnosis.ts 1086~1087행, ai-visibility-report.ts 1141/1481행, ai-visibility-html-report*.ts 다수
- 현재: "잠재 환자" → "웹사이트 유입 누락 환자"로 치환
- 수정: 문맥에 맞게 다양한 표현 사용
  - 표지/KPI: "예상 미유입 잠재 환자" 
  - 본문 경고: "매월 약 N명의 잠재 환자가 웹사이트를 통해 유입되지 못하고 있습니다"
  - 이중표현 방지: "누락 환자가 이탈" → "잠재 환자가 유입되지 못하고 있습니다"
